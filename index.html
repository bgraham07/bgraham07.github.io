Module 3: Security-Enhancement Refactor

This module focuses on enhancing the security of my application and refactoring the architecture to follow best practices. The updated code is hosted in a separate branch of my GitHub repository.

  • GitHub Branch: module3-refactor
  • Key Enhancements:
    • Frontend login moved from Handlebars to Angular SPA
    • Backend JWT-based authentication middleware added
    • All code commented and documented for clarity
    • Narrative:

      This enhancement focused on implementing full authentication in my Travlr Gateways full stack app using passport.js. It included hashed password validation, secure session handling, and route protection via express-jwt. This process emphasized the importance of secure backend design and demonstrated my ability to manage sessions securely. I initially only implemented this on Handlebars and not Angular SPA, which I later identified as a gap to improve on.

Module 5: Deployment and Continuous Integration

This module covers the deployment process and setting up continuous integration pipelines to automate testing and deployment. It ensures the application is reliably updated and maintained.

  • GitHub Branch: module5-refactor
  • Key Enhancements:
    • Improved data handling across both admin and public-facing views
    • Implemented rate limiting for DB read/write operations
    • Secured MongoDB connection settings and pool configuration
    • Applied Joi-based validation for inputs and ObjectIds
    • Hardened error handling for duplicate keys, cast errors, and validation failures
    • Sanitized queries to prevent NoSQL injection attacks
    • Narrative:

      In this module, I worked on enhancing database security and reliability in my full stack app. I implemented rate limiting using a custom rateLimiter.js, optimized MongoDB connection handling in db.js, and enforced timeouts. Additionally, I introduced Joi validation, robust error handling, and query sanitization to defend against NoSQL injection. This helped me treat the project as a real-world system requiring secure and scalable practices.
